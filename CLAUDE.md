# MirrorPlay 鏡像書寫英文學習工具

## 專案概述

一個為 ADHD 使用者設計的英文鏡像書寫學習小工具。核心概念：將英文文字左右翻轉成鏡像，使用者透過「解密」鏡像文字並手寫出正確英文的過程來學習，兼具專注力訓練和符號認知體驗。

**設計哲學：這是一個解密遊戲，不是考試工具。不做自動批改、不做分數系統、不做任何讓使用者有壓力的功能。**

## 目標使用者

- ADHD 特質：喜歡從結果看回開頭，需要趣味性維持注意力
- 有電繪筆的使用者（iPad / 繪圖板）
- 不追求「學會英文」，而是透過可控的加密解碼過程提升符號認知
- 使用繁體中文介面

## 核心流程

```
上傳圖片（含英文文字）
    ↓
系統 OCR 辨識圖片中所有英文單字
    ↓
┌─────────────────┬──────────────────┐
│    左側區域       │    右側區域        │
│  顯示鏡像翻轉的   │  生成對應的填空     │
│  英文文字         │  手寫欄位          │
│  （右讀左）       │  （用電繪筆書寫）    │
└─────────────────┴──────────────────┘
    ↓
使用者看左邊鏡像 → 在右邊寫出正確英文
    ↓
寫完後點擊單字 → 顯示：
  1. 對應圖片（視覺化意思）
  2. 中文翻譯
  3. 使用者手寫中文意思的欄位
    ↓
如果是片語/複雜用法 → 呼叫 AI 提供語境註解
```

## 功能規格

### 必要功能（MVP）

1. **圖片上傳**
   - 支援 jpg / png / webp
   - 上傳後原圖保留（使用者自行對照用，不需要顯示在主介面）

2. **OCR 辨識**
   - 辨識圖片中所有英文文字
   - 技術方案：Tesseract.js（純前端，不需後端）
   - 輸出為單字陣列，保留原始順序

3. **左側鏡像顯示區**
   - 將辨識出的英文文字水平翻轉（CSS transform: scaleX(-1)）
   - 例如 "apple" 顯示為 "elppa"（鏡像，不是單純反轉字串）
   - 保持原文的排列順序和段落結構

4. **右側手寫填空區**
   - 每個單字對應一個空白手寫欄位
   - 使用 HTML Canvas 接收電繪筆輸入
   - 欄位大小根據單字長度自適應
   - 支援清除重寫

5. **寫完後的學習延伸**
   - 點擊已完成的單字可展開：
     - 圖片：串接免費圖片搜尋 API（如 Unsplash API 或 Pexels API）
     - 中文翻譯：串接免費翻譯 API
     - 手寫中文欄位：讓使用者手寫中文意思

### 需要 AI 的功能

6. **片語 / 語句註解**（僅在使用者主動請求時觸發）
   - 當辨識到片語（如 "come to one's rescue"）時提供：
     - 整體意思解釋
     - 使用語境範例
   - 技術方案：呼叫 Claude API（Sonnet 即可，成本低）
   - 不主動跳出，使用者點「解釋」按鈕才觸發

### 防躁小畫板

7. **塗鴉畫板**（常駐功能）
   - 一個可收合/展開的小畫板，固定在介面角落或側邊
   - 白底黑筆，最基礎的塗鴉功能
   - 用途：使用者在解密過程中注意力飄走時，可以隨手畫幾筆再回來，不需要離開 app
   - 功能：畫筆（黑色）、橡皮擦、清除全部，僅此三個
   - 不儲存、不匯出、不跟學習流程連動——純粹是一個讓手動起來的地方
   - 技術：獨立的 HTML Canvas，跟手寫填空區分開
   - 收合時顯示為一個小圖示（如 🎨），展開後覆蓋部分畫面或滑出側邊欄
   - **設計原則：這不是功能，是安全閥。ADHD 的注意力需要出口，與其跳出 app 去別的地方塗鴉，不如在 app 內提供一個不帶任何目的的空白空間。**

### 明確不做的功能

- ❌ 自動比對手寫答案（不是考試）
- ❌ 分數 / 等級 / 排行榜
- ❌ 學習進度追蹤
- ❌ 單字拆字母填空（整個單字為最小單位）
- ❌ 任何讓人有壓力的遊戲化機制

## 技術架構

### 前端（純前端 Web App）

- **框架**：React（單頁應用）
- **樣式**：Tailwind CSS
- **OCR**：Tesseract.js（瀏覽器端執行）
- **手寫**：HTML Canvas API + Pointer Events（支援電繪筆壓感）
- **圖片翻轉**：CSS transform

### 外部 API（按需呼叫）

- **翻譯**：免費翻譯 API（Google Translate 免費層 或替代方案）
- **圖片**：Unsplash API 或 Pexels API（免費額度）
- **AI 註解**：Claude API（Sonnet 模型，僅片語解釋時呼叫）

### 部署

- 靜態網站，可部署到 GitHub Pages / Vercel / Netlify
- 不需要後端伺服器（API 呼叫從前端直接發出，或用 serverless function 包一層保護 API key）

## UI 設計方向

- 簡潔、低刺激、不花俏
- 深色模式優先（長時間使用不刺眼）
- 左右分割佈局，中間清楚分隔
- 手寫區域要夠大，給電繪筆足夠的書寫空間
- 繁體中文介面

## 圖片來源場景

使用者可能上傳的圖片類型：
- 英文課本頁面（如照片中的英文學習本）
- 歌詞截圖
- 社群貼文截圖
- 書籍內頁
- 任何含有英文文字的圖片

## 開發優先順序

1. **Phase 1**：上傳圖片 → OCR → 鏡像顯示 → 手寫填空（核心迴圈）
2. **Phase 2**：單字點擊展開圖片 + 中文翻譯 + 手寫中文
3. **Phase 3**：片語 AI 註解
4. **Phase 4**：UI 優化、深色模式、電繪筆體驗優化

## 注意事項

- 使用者是繁體中文使用者，所有介面文字和翻譯結果都用繁體中文
- 鏡像是視覺上的水平翻轉（CSS scaleX(-1)），不是程式上的字串反轉。"apple" 鏡像後在視覺上看起來像手寫的鏡像字，不是 "elppa" 這個字串
- 這個工具的核心價值是「解密過程」本身，不是學習成效。不要加任何量化學習的功能
